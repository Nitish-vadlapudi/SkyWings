Create a modern airline booking and check-in web application named Sky Wings. The application should begin with an animated Sky Wings logo splash screen, after which users are directed to an authentication page. The authentication page must provide three options: Sign Up for new users, Sign In for existing users, and Admin Login. New users should register by entering their full name, age, gender, phone number, email address, and creating a password. Existing users should be able to log in using their registered email address and password. An Admin Login option should also be available on the same page, with the admin password set to NJSW05. Upon successful admin login, the administrator should gain access to a dedicated admin dashboard. The application must implement secure authentication, password encryption, data validation, session management, and role-based access control to ensure users can access only their own information while administrative functions remain restricted.

After successful user login or registration, users should be taken to a dashboard displaying two main options: Browse Flights and Start Check-In. The Browse Flights section should display all available flights along with details such as flight name, flight number, source, destination, departure time, arrival time, ticket price, available travel classes, and seat availability. Users should be able to search, filter, and sort flights by source, destination, date, class, and price. When a user selects a flight, a detailed flight information page should open showing complete flight details, travel class options, and real-time seat availability. Seat availability should be indicated using color codes: green for high availability, yellow for limited seats, and red for no seats available. The application must not automatically assign seats. Instead, users should be able to view an interactive seat map and manually select their preferred seats. The seat map should clearly display occupied and available seats. Multiple seating categories must be available, including First Class, Business Class, Premium Economy, and Economy Class, with each class displaying its ticket price, available seats, and class-specific benefits.

Users should be able to book one or multiple tickets in a single reservation, allowing family, group, or individual bookings depending on seat availability. During booking, passengers must provide their name, age, and gender for each ticket. After successful booking, the system should generate a unique PNR number, booking reference ID, and assigned seat numbers. A booking confirmation page should then display complete passenger details, email address, flight details, travel class, seat numbers, booking date and time, total number of tickets, fare breakdown, taxes, and total amount paid. The application must not automatically redirect users to the Check-In page after booking; instead, it should remain on the booking confirmation screen and allow users to choose their next action.

After booking, the system should automatically generate a professional digital e-ticket and a downloadable PDF ticket. The ticket should contain the Sky Wings branding, passenger details, flight information, travel class, seat numbers, booking reference, PNR number, booking summary, and a QR code for verification. Users should be able to view, print, or download the ticket at any time. Additionally, a confirmation email should be sent to the registered email address containing passenger details, flight information, PNR number, booking summary, and the PDF ticket as an attachment. The ticket PDF should have a professional airline-style layout and include all essential travel information.

The Start Check-In feature should be accessible separately from the dashboard. Users should be able to enter their PNR number to retrieve booking details and complete the check-in process. Once check-in is completed, the system should generate a digital boarding pass containing the passenger’s name, flight number, PNR number, travel class, seat number, boarding gate, boarding time, departure time, arrival time, and a QR code. Users should be able to download, print, and save the boarding pass in PDF format for travel.

The Admin Dashboard should be accessible only to the administrator and must include comprehensive flight, booking, and passenger management capabilities. The administrator should be able to view all registered users, search user records, monitor user activity, view all bookings, search passenger records, monitor booking history, and track generated PNR numbers. The admin should also be able to add new flights by entering flight name, flight number, source, destination, departure time, arrival time, total seat capacity, class-wise seat allocation for First Class, Business Class, Premium Economy, and Economy, along with ticket pricing for each class. Additionally, the administrator should be able to edit flight details, update schedules, modify seat availability, manage ticket pricing, and delete flights when necessary. The dashboard should also include booking management features such as viewing all reservations, modifying bookings, canceling reservations, tracking seat occupancy, monitoring revenue, and generating booking and occupancy reports.

The application should feature a modern aviation-inspired user interface using sky blue, white, and navy blue colors, smooth animations, responsive layouts for desktop and mobile devices, intuitive navigation, advanced flight search and filtering, interactive seat selection, loading indicators, toast notifications, and professional success and error messages. The final application should provide a complete airline reservation ecosystem including user registration, secure authentication, flight browsing, manual seat selection, multi-passenger booking, digital ticket generation, PDF downloads, QR-code-based boarding passes, check-in management, and a powerful admin panel comparable to modern commercial airline booking systems.
